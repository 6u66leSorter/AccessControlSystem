# Access Control System

Автоматизированное рабочее место охранника КПП
«Пропускной пункт предприятия»

## Описание проекта

Данный проект представляет собой прототип автоматизированного рабочего места охранника контрольно-пропускного пункта (КПП) предприятия.
Система предназначена для учета доступа сотрудников, посетителей и транспортных средств на территорию организации, а также для фиксации всех входов и выходов в журнале доступа.

Проект разработан в рамках курсовой работы по дисциплине «Управление качеством программного обеспечения» с применением принципов управления тестами и модульного тестирования в среде NUnit.

---

## Основные функции системы

Реализованы следующие ключевые функции:

* допуск сотрудников по электронным картам доступа;
* регистрация отказов в доступе с указанием причины;
* учет въезда и выезда автотранспорта предприятия и сторонних организаций;
* учет посетителей (вход и выход);
* ведение журнала доступа (Access Log);
* получение списков активных посетителей и транспортных средств.

Часть функций (учет ключей, материальных ценностей, расширенные отчеты) предусмотрены архитектурно и могут быть реализованы в дальнейшем.

---

## Архитектура проекта

Проект реализован с использованием слоистой архитектуры:

* **Entities** — доменные сущности (Employee, Visitor, VehiclePass, AccessLog);
* **Repositories** — интерфейсы доступа к данным;
* **Services** — бизнес-логика системы контроля доступа;
* **Tests** — unit-тесты бизнес-логики;
* **Demo Console App** — консольное приложение для демонстрации работы системы.

Бизнес-логика изолирована от слоя хранения данных, что обеспечивает тестируемость и расширяемость системы.

## Unit-тестирование

Для ключевых бизнес-сценариев разработаны unit-тесты с использованием NUnit.
Тесты выполняются без подключения базы данных за счет использования mock-репозиториев.

Unit-тестами покрыты следующие сценарии:

* допуск и отказ сотрудникам;
* регистрация и завершение визитов транспорта;
* регистрация и завершение визитов посетителей;
* обработка некорректных сценариев.

## Запуск проекта

1. Клонировать репозиторий:

   ```
   git clone <repository-url>
   ```

2. Открыть решение в Visual Studio.

3. Запустить unit-тесты:

   ```
   dotnet test
   ```

4. При необходимости запустить демонстрационное консольное приложение.
